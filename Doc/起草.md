## 一、产品定位与命名

**名称建议**：`MirrorPilot`（镜像领航员）或 `RegHub`（Registry Hub）或 `PkgPilot`

**定位**：一个桌面 GUI 工具，统一管理开发环境中各包管理器的镜像源配置，附带网络测速和本地缓存管理功能。

## 二、核心功能模块

### 模块 1：镜像源管理（核心）

#### 1.1 支持的包管理器

| 包管理器 | 配置文件/环境变量 | 平台 |
|---------|------------------|------|
| **npm** | `.npmrc` → `registry` | 全平台 |
| **yarn** | `.yarnrc` → `registry` | 全平台 |
| **pnpm** | `.npmrc` → `registry` | 全平台 |
| **pip** | `pip.ini` / `pip.conf` → `index-url` | 全平台 |
| **uv** | `uv.toml` / 环境变量 | 全平台 |
| **Go** | `go env` → `GOPROXY` | 全平台 |
| **Maven** | `settings.xml` → `<mirrors>` | 全平台 |
| **Gradle** | `init.gradle` / `gradle.properties` | 全平台 |
| **Docker** | `daemon.json` → `registry-mirrors` | 全平台 |
| **apt** | `/etc/apt/sources.list` + `/etc/apt/sources.list.d/` | Linux |
| **yum/dnf** | `/etc/yum.repos.d/*.repo` | Linux |
| **Homebrew** | 环境变量 `HOMEBREW_BREW_GIT_REMOTE` | macOS/Linux |
| **Tauri** | 实际是通过 npm/cargo，可归到 npm 里 | - |
| **Cargo** | `config.toml` → `source.crates-io` | 全平台 |
| **NuGet** | `NuGet.Config` | Windows |
| **Chocolatey** | `chocolatey.config` | Windows |

#### 1.2 源地址数据结构

```yaml
Source:
  id: string           # 唯一标识
  name: string         # 显示名称（如"清华镜像"）
  url: string          # 源地址
  type: string         # 包管理器类型（npm/pip/go...）
  is_builtin: boolean  # 是否内置
  is_custom: boolean   # 是否用户自定义
  region: string       # 地区（cn/us/eu...）
  status: string       # 状态（active/inactive/testing）
  latency: number      # 延迟（ms）
  speed: number        # 速度（KB/s）
  last_tested: datetime # 上次测试时间
```

#### 1.3 内置源地址库

每个包管理器预设 5-10 个常用镜像：

- **npm**: npm官方、淘宝、腾讯云、华为云、清华、中科大
- **pip**: PyPI官方、清华、阿里云、豆瓣、华为云、中科大
- **Go**: 官方、Goproxy.cn、阿里云、七牛云
- **Maven**: Maven Central、阿里云、华为云、网易
- **Docker**: Docker Hub官方、阿里云、腾讯云、DaoCloud、清华
- **apt**: 各发行版官方源 + 国内镜像（阿里云、清华、中科大）
- **yum**: 各发行版官方源 + 国内镜像

#### 1.4 交互流程

```
用户选择包管理器（如 npm）
    ↓
展示当前已配置的源（高亮当前生效源）
    ↓
展示内置源列表（带延迟和速度标签）
    ↓
用户点击"应用" → 修改配置文件
    ↓
用户点击"测试" → 测速并更新标签
    ↓
用户点击"添加自定义" → 弹窗输入名称+URL → 保存到本地数据库
    ↓
用户点击"删除" → 仅可删除自定义源
```

### 模块 2：网络测速

#### 2.1 测速方式

| 方式 | 说明 |
|------|------|
| **HTTP Ping** | 对源地址发 HEAD 请求，计算 RTT |
| **下载测速** | 下载一个固定大小文件（如 1MB 的测试包），计算实际速度 |
| **并发测速** | 同时测试多个源，对比排序 |

#### 2.2 测速策略

- **快速检测**：只测延迟（RTT），1-2 秒出结果
- **完整测速**：延迟 + 实际下载速度，5-10 秒
- **后台自动测速**：定时刷新（可选，默认关闭）

#### 2.3 结果展示

- 延迟：绿色(<100ms) / 黄色(100-500ms) / 红色(>500ms)
- 速度：柱状图或数字
- 推荐标签：自动推荐最快的源

### 模块 3：本地包缓存管理

#### 3.1 扫描的缓存目录

| 包管理器 | 默认缓存路径 |
|---------|-------------|
| npm | `~/.npm` 或 `%LOCALAPPDATA%\npm-cache` |
| yarn | `~/.cache/yarn` |
| pnpm | `~/.pnpm-store` |
| pip | `~/.cache/pip` |
| uv | `~/.cache/uv` |
| Go | `~/go/pkg/mod` |
| Maven | `~/.m2/repository` |
| Gradle | `~/.gradle/caches` |
| Cargo | `~/.cargo/registry/cache` |
| Docker | `/var/lib/docker` (Linux) / WSL2 或 Docker Desktop 指定路径 |
| apt | `/var/cache/apt/archives` |
| yum | `/var/cache/yum` |

#### 3.2 功能

- **扫描**：递归扫描缓存目录，统计大小、文件数
- **查看**：树形结构展示，支持搜索
- **清理**：
  - 单个包管理器清理
  - 一键清理所有
  - 按时间清理（如"删除 30 天未使用的"）
- **大小统计**：饼图/柱状图展示各包管理器占用比例

## 四、数据流设计

### 4.1 应用源流程

```
用户点击"应用此源"
    ↓
Frontend: 调用 IPC command `apply_source`
    ↓
Backend: 
  1. 验证 URL 格式
  2. 根据包管理器类型，确定配置文件路径
  3. 备份原配置（可选）
  4. 修改配置文件（如 .npmrc）
  5. 验证修改是否成功（读取确认）
  6. 更新数据库中的当前源记录
    ↓
返回结果 → Frontend 显示成功/失败提示
```

### 4.2 测速流程

```
用户点击"开始测速"
    ↓
Frontend: 调用 IPC command `test_sources`
    ↓
Backend:
  1. 获取选中源列表
  2. 创建异步任务池（限制并发数，如 5 个）
  3. 对每个源：
     - 发送 HEAD 请求，记录 RTT
     - 下载测试文件（如 1MB），记录速度
  4. 实时通过 Event 推送进度到 Frontend
  5. 汇总结果，更新数据库
    ↓
Frontend: 实时更新进度条和结果列表
```

### 4.3 扫描缓存流程

```
进入"本地缓存"页
    ↓
Frontend: 调用 IPC command `scan_cache`
    ↓
Backend:
  1. 遍历配置的缓存路径
  2. 统计每个目录的大小和文件数
  3. 返回结构化数据
    ↓
Frontend: 渲染饼图和列表
```

## 五、关键设计决策

### 5.1 配置文件修改方式

| 方案 | 优点 | 缺点 |
|------|------|------|
| **直接修改文件** | 标准、通用 | 需处理格式、权限、路径差异 |
| **调用 CLI 命令** | 安全、官方支持 | 不是所有工具都提供（如 apt 没有改源命令） |
| **混合方案** | 灵活 | 复杂度高 |

**建议**：优先用 CLI 命令（如 `npm config set registry`），不支持 CLI 的再直接改文件（如 apt 的 sources.list）。

### 5.2 权限处理

- **Windows**：大部分配置文件在用户目录，无需提权
- **macOS/Linux**：apt/yum/Docker 配置可能需要 sudo
  - 方案 A：检测到需要权限时，弹出密码输入框
  - 方案 B：提供命令复制功能，让用户手动 sudo 执行
  - **推荐 B**：更安全，避免存储密码

### 5.3 自定义源存储

- 内置源：硬编码在 Rust 代码或 JSON 资源文件中
- 自定义源：SQLite 数据库（`~/.mirrorpilot/sources.db`）
- 用户配置：JSON 文件（`~/.mirrorpilot/config.json`）

### 5.4 跨平台路径处理

```rust
// Rust 中使用 dirs 库获取跨平台路径
let home = dirs::home_dir()?;           // C:\Users\Name 或 /home/name
let config_dir = dirs::config_dir()?;   // AppData\Roaming 或 ~/.config
let cache_dir = dirs::cache_dir()?;     // AppData\Local 或 ~/.cache
```
